1. [static 키워드]에 대해서 설명하세요. 

	static 키워드는 클래스 멤버로 만들어준다.
	클래스 멤버는  Method Area 에 생성되고 
	객체를 만들때마다 생성되지 않고 한번 메모리에 올라간다.
	그리고 모든 객체가 공유하게 된다.
	예를 들어 rate라는 이자율은 어느 객체든 동일하기 때문에 static을 붙힐 수 있다.

2.  is -a 관계, 상속(inheritance)에 대해서 설명하세요 

	상속이란 기존의 클래스를 재사용해 새로운 클래스를 작성하는 것이다.
	코드를 재사용해 생산성이 향상되고 유지보수가 용이해진다.
	상속 선언 형식
	[접근지정자][기타제어자] class 새클래스명 [extends 기존클래스명] { } 

	상속 관계 용어
	기존 클래스명 = 부모 클래스, super클래스
	새클래스명 = 자식 클래스, sub클래스

4. 오버로딩(overload)과 오버라이딩(override)에 대해서 설명하세요 .
    1) 오버로딩         
	함수 이름은 같고 매개변수의 타입이나 갯수가 다른 함수를 중복해서 만드는 것이다.
	
    2) 오버라이딩 
	상속관계에서 부모로부터 물려받은 메서드를 재정의 하는 것이다.	

4-2. 오버라이딩 조건에 대해서 설명하세요.
	1. 함수명이 같아야함
	2. 매개변수 갯수와 타입이 같아야함
	3. 리턴타입이 같아야함
	4. 접근지정자는 부모와 같거나 더 넓게 지정해야한다.
	5. 부모보다 더 많은 예외를 선언할 수 없다.
	6. 인스턴스 메서드 <-> 클래스 메서드 로 변경 불가능하다.

5. 단일 상속과 다중 상속에 대해서 설명하세요 .

	단일 상속은 오직 한 개의 클래스만 상속받는 것이다.
	만약 다중 상속처럼 하고 싶다면 인터페이스를 이용한다.
	
	다중 상속은 여러 개의 클래스를 상속받는 것이다.
	하지만 여러 개의 클래스를 상속 받으면 멤버가 같으면 충돌나기 때문에
	자바에서는 지원하지 않는다.

6. [@Override] 애노테이션(Annotation)에 대해서 설명하세요.

	@Override는 단지 부모로부터 물려받은 메서드를 재정의할 때
	오버라이딩 조건이 맞는지 확인한다.

9. 참조형( 클래스 )의 형변환에 대해서 설명하세요. 

	// 업 캐스팅(자식 클래스 -> 부모 클래스)
	Employee emp = new Regular();
	원래는 Employee emp = (Employee)new Regular(); 라고 해야되지만 업 캐스팅은 
	자동으로 형변환이 된다.

	// 다운 캐스팅(부모 클래스 -> 자식 클래스)
	다운 캐스팅은 자식 클래스가 부모 클래스의 멤버보다 같거나 더 많을거기 때문에
	자동 형변환이 되지 않고 강제로 형변환 해야한다.
	Employee emp = new Regular();
	Regular emp2 = (Regular)emp;
	
	아래 코드는 에러가 난다. 실제 인스턴스는 부모여서 자식 객체로 형변환 할 수 없다.
	Regular emp = (Regular)new Employee();

12. final 키워드 설명
  1) 클래스 앞의 final 붙인 경우

	자식 클래스를 생성할 수 없는 클래스가 된다.	

  2) 메서드 앞에 final 붙인 경우

	오버라이딩이 불가능한 메서드가 된다.

  3) 변수 앞에 final 붙인 경우

	상수가 된다.
